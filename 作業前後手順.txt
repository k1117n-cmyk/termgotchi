  - 作業前: git pull
  - 作業後: git add → git commit → git push

Git Lesson1

  1. 状態を確認する

  cd ~/termgotchi
  git status

  2. 今回コミットしたいファイルだけ追加する

  git add README.md README.ja.md

  作業前後手順.txt は今回は含めないので add しません。

  3. 追加結果を確認する

  git status

  Changes to be committed に README.md と README.ja.md が出れば正しいです。

  4. コミットする

  git commit -m "Add Japanese README"

  5. push する

  git push

  6. 最後に確認する

  git status

  nothing to commit, working tree clean なら完了です。

  では次は、「何をコミットしたか」を読む練習です。

  まず直近の履歴を短く見ます。

  git log --oneline -5

  見方:

  - 左側の短い文字列: コミットID
  - 右側: コミットメッセージ
  - 上にあるほど新しい

  次に、最新コミットの中身を見ます。

  git show --stat HEAD

  見方:

  - HEAD は「いまの最新コミット」
  - どのファイルが変更されたか
  - 何行増減したか
  - コミットメッセージと日時も分かる

  さらに内容まで見たいなら:

  git show HEAD

  これは差分そのものを表示します。最初は少し長いので、まずは --stat 付きで見
  るのが楽です。

  練習の流れ:

  1. git log --oneline -5
  2. git show --stat HEAD
  3. 余裕があれば git show HEAD

  出てきた結果で、

  - 最新コミットが何か
  - どのファイルが入ったか
    を自分で読めれば十分です。必要なら、その出力の読み方を次にこちらで一緒に
    確認します。
